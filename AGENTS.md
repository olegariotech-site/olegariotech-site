# AGENTS.md — Regras do Projeto Olegario Tech

Este repositório é o site oficial da Olegario Tech.

## Fonte oficial
- Repositório: `olegariotech-site/olegariotech-site`
- Domínio: `https://olegariotech.com.br/`
- Publicação: GitHub Pages em `main` / root.

## Regra principal
Não recriar o site do zero. Preservar o que funciona e evoluir por partes.

## Nunca apagar sem auditoria
- `CNAME`
- `favicon.ico`
- `robots.txt`
- `sitemap.xml`
- `README.md`
- assets de eBooks e produtos digitais
- imagens usadas por SEO/OG
- `assets/audio/background.mp3`
- seções de produtos digitais/eBooks
- apresentação da Olegario Tech
- WhatsApp principal
- schema LocalBusiness
- Three.js / rede neural / fundo animado

## Fluxo obrigatório
1. Criar branch antes de mexer.
2. Fazer commits pequenos.
3. Não alterar `main` diretamente sem autorização explícita.
4. Testar HTML, CSS e JavaScript.
5. Conferir mobile.
6. Conferir links de WhatsApp.
7. Conferir que eBooks e produtos digitais continuam visíveis.
8. Só limpar arquivos antigos depois do deploy aprovado.

## Direção visual
A Olegario Tech deve parecer uma agência boutique digital:
- escura
- premium
- tecnológica
- autoral
- com profundidade
- orientada a conversão
- focada em negócios locais de Valinhos, Campinas e região

## Portfólio
A seção `#portfolio` deve ser um `portfolio-showcase` imersivo em scroll. Não deve voltar a ser uma simples grade 2x2, salvo decisão explícita do Alexandre.
